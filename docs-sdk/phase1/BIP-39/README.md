## Overview

BIP-39 defines a standard for generating human-readable mnemonic phrases
from cryptographically secure entropy. A mnemonic is the primary backup
mechanism for HD wallets - anyone with the phrase can reconstruct the
entire wallet.

The process:
```
Entropy (random bytes)
    -> + SHA256 checksum
Bits (entropy + checksum)
    -> split into groups of 11
Word indices (0-2047)
    -> lookup in wordlist
Mnemonic phrase
```

**Official source:**
[BIP-39](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki)


## Decision 1: Five Components Instead of One Class

**Why:**
BIP-39 has five naturally distinct responsibilities. Splitting them
into five classes follows the Single Responsibility Principle and
makes each component independently testable and reusable.

| Class | Responsibility |
|---|---|
| `MnemonicStrength` | Defines valid entropy sizes (128-256 bits) |
| `Entropy` | Generates and validates random entropy bytes |
| `Wordlist` | Abstract contract for any wordlist (any language) |
| `WordlistEn` / `WordlistEs` | Official BIP-39 words per language |
| `Mnemonic` | Converts entropy → words and words → entropy |

**Implementation:**
```dart
// Each component is independently usable
final entropy = Entropy.generate(strength: MnemonicStrength.words24);
final bits    = Entropy.toBits(entropy);
final index   = bits.take(11).fold(0, (acc, b) => (acc << 1) | b);
final word    = WordlistEn.instance.wordAt(index);
```


## Decision 2: MnemonicStrength as Enum with Computed Properties

**Why:**
BIP-39 defines exactly 5 valid entropy sizes. An enum prevents invalid
values at compile time and centralizes the formula logic. Each variant
computes `bytes`, `checksumBits`, `totalBits`, and `wordCount`
directly from `bits` per BIP-39 formulas:
- `CS = ENT / 32`
- `MS = (ENT + CS) / 11`

**Official source:**
[BIP-39 - Generating the mnemonic](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki#generating-the-mnemonic)

**Implementation:**
```dart
enum MnemonicStrength {
  words12(128),
  words15(160),
  words18(192),
  words21(224),
  words24(256);

  const MnemonicStrength(this.bits);
  final int bits;
  int get bytes        => bits ~/ 8;
  int get checksumBits => bits ~/ 32;
  int get totalBits    => bits + checksumBits;
  int get wordCount    => totalBits ~/ 11;
}
```

**Verification:**
```dart
test('CS = ENT / 32 holds for all strengths', () {
  for (final s in MnemonicStrength.values) {
    expect(s.checksumBits, equals(s.bits ~/ 32));
  }
});
```


## Decision 3: SHA256 Checksum via pointycastle

**Why:**
BIP-39 requires a checksum of `ENT/32` bits taken from the first byte(s)
of `SHA256(entropy)`. `pointycastle` is already a dependency for
secp256k1 - reusing `SHA256Digest` avoids adding a new dependency.

**Official source:**
[BIP-39 - Generating the mnemonic](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki)

> "A checksum is generated by taking the first ENT/32 bits of its
> SHA256 hash."

**Implementation:**
```dart
static int computeChecksum(Uint8List entropy) {
  validate(entropy);
  final hash = _sha256(entropy);
  final strength = _strengthFromBytes(entropy.length);
  return hash[0] >> (8 - strength.checksumBits);
}
```

**Verification (runtime):**
```dart
test('all-zeros 128-bit entropy has checksum 3', () {
  // SHA256 of 16 zero bytes → first byte = 0x66 = 01100110
  // First 4 bits = 0011 = 3 (verified from runtime output)
  final entropy = Uint8List(16);
  expect(Entropy.computeChecksum(entropy), equals(3));
});
```


## Decision 4: Wordlist as Abstract Class with Binary Search

**Why:**
`Wordlist` is an abstract class (not an interface) because:
1. It provides a concrete `indexOf()` using binary search - valid
   because BIP-39 requires wordlists to be sorted alphabetically
2. Subclasses only need to provide the `words` list and `language`
3. Any future language can extend `Wordlist` without duplicating logic

Binary search is O(log n) vs O(n) for `List.indexOf()`. For 2048 words,
binary search needs at most 11 comparisons.

**Official source:**
[BIP-39 - Wordlists](https://github.com/bitcoin/bips/blob/master/bip-0039.mediawiki#wordlist)

> "c) sorted wordlists - allows for more efficient lookup"

**Implementation:**
```dart
int indexOf(String word) {
  var low = 0;
  var high = words.length - 1;
  while (low <= high) {
    final mid = (low + high) ~/ 2;
    final comparison = words[mid].compareTo(word);
    if (comparison == 0) return mid;
    if (comparison < 0) { low = mid + 1; }
    else { high = mid - 1; }
  }
  return -1;
}
```


## Decision 5: WordlistEn - Singleton + static const

**Why:**
The English wordlist is immutable and identical across all usages.
A singleton prevents multiple 2048-element lists in memory.
`static const` ensures the list lives in the Dart constant pool
and is never re-allocated.

**Source:**
[bitcoin/bips - english.txt](https://github.com/bitcoin/bips/blob/master/bip-0039/english.txt)

**Implementation:**
```dart
class WordlistEn extends Wordlist {
  static final WordlistEn instance = WordlistEn._();
  WordlistEn._();

  @override
  String get language => 'en';

  @override
  List<String> get words => _words;

  static const List<String> _words = <String>[
    'abandon', 'ability', 'able', ... // 2048 words
  ];
}
```

**Verification:**
```dart
test('first word is "abandon" (index 0)', () {
  expect(WordlistEn.instance.wordAt(0), equals('abandon'));
});
test('last word is "zoo" (index 2047)', () {
  expect(WordlistEn.instance.wordAt(2047), equals('zoo'));
});
```


## Decision 6: WordlistEs - HashMap + NFKD + NFC Override

**Why:**
The Spanish wordlist introduces three challenges not present in English:

**Challenge 1 - NFKD encoding:**
The official BIP-39 Spanish file stores words in NFKD form (combining
characters). `'ábaco'` is stored as `'a' + U+0301 + 'baco'`, not as
`U+00E1 + 'baco'`. Binary search with `compareTo()` would fail because
Dart compares by unicode code point, not by combining character sequences.

**Challenge 2 - Spanish alphabet ordering:**
The Spanish wordlist uses the traditional Spanish alphabet where `ñ`
is a distinct letter that comes after `n`. This creates 11 pairs that
are out of unicode order (e.g. `'anuncio'` → `'añadir'`). Binary search
is invalid for this ordering.

**Challenge 3 - Accent-insensitive lookup:**
Per BIP-39 spec, users without a Spanish keyboard must be able to type
`'abaco'` and find `'ábaco'` at index 0.

**Solution:**
1. Store words in NFKD (as in the official file)
2. Override `wordAt()` to return NFC (human-readable form)
3. Override `indexOf()` with HashMap O(1) - keyed by both NFC and
   accent-stripped forms
4. Override `validate()` to skip unicode sort check

**Official source:**
[bitcoin/bips - spanish.txt](https://github.com/bitcoin/bips/blob/master/bip-0039/spanish.txt)

> "Special Spanish characters like 'ñ', 'á', etc. are considered
> equal to 'n', 'a', etc. in terms of identifying a word."

**Implementation:**
```dart
@override
String wordAt(int index) {
  final nfkd = super.wordAt(index);
  return _toNfc(nfkd); // Convert NFKD → NFC for callers
}

@override
int indexOf(String word) {
  final nfc = _toNfc(word.toLowerCase());
  return _index[nfc] ?? _index[_strip(nfc)] ?? -1;
}

static String _toNfc(String word) {
  return word
      .replaceAll('a\u0301', 'á')
      .replaceAll('n\u0303', 'ñ'); // + other combining chars
}

static String _strip(String word) {
  return word
      .replaceAll('á', 'a')
      .replaceAll('ñ', 'n'); // + other accents
}
```

**Verification:**
```dart
test('finds "ábaco" with unaccented form "abaco"', () {
  expect(WordlistEs.instance.indexOf('abaco'), equals(0));
});
test('"anuncio" appears before "añadir"', () {
  expect(WordlistEs.instance.indexOf('anuncio'),
      lessThan(WordlistEs.instance.indexOf('añadir')));
});
```


## Decision 7: Mnemonic.toString() Always Returns [REDACTED]

**Why:**
A mnemonic phrase is equivalent in sensitivity to a private key.
If a developer accidentally logs a `Mnemonic` instance, the phrase
must never appear in logs, stack traces, or error messages.

**Implementation:**
```dart
@override
String toString() => 'Mnemonic[REDACTED]';
```

**Security test:**
```dart
test('toString() does not contain any word from the phrase', () {
  final mnemonic = Mnemonic.generate();
  for (final word in mnemonic.words) {
    expect(mnemonic.toString().contains(word), isFalse);
  }
});
```

> `phrase` getter is the intentional way to access the actual phrase.


## Decision 8: Checksum Verification on fromPhrase()

**Why:**
When importing an existing phrase, the BIP-39 checksum must be
verified. A wrong word or wrong word order should fail immediately
with a clear error, not silently produce an incorrect wallet.

**Implementation:**

## Summary

| Decision | Choice | Rationale |
|---|---|---|
| Architecture | 5 classes | Single responsibility, independent testing |
| Entropy sizes | `MnemonicStrength` enum | Compile-time safety, formulas centralized |
| Checksum | SHA256 via pointycastle | Reuse existing dependency |
| Base class | `abstract class Wordlist` | Contract + binary search shared |
| EN lookup | Binary search O(log n) | List is sorted per BIP-39 spec |
| ES storage | NFKD (official file format) | Per BIP-39 spec requirement |
| ES output | NFC via `wordAt()` override | Human-readable for callers |
| ES lookup | HashMap O(1) | Binary search invalid for Spanish ñ ordering |
| ES input | Accent-insensitive | Per BIP-39 spec for Spanish |
| Security | `toString()` = `[REDACTED]` | Mnemonic = private key sensitivity |
| Import | Checksum verified | Fail fast on typos or wrong order |
